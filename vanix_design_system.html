1 — Color tokens

Brand — two-hue system only. No other hue.

brandGreen #4DDE95 Logo color. Accent only. Never on white — fails WCAG AA (2.5:1). Use on #111 or on #E8F5EE pill bg.
greenDeep #2EBD7E CTA button fill (Continue / Confirm) with dark #111 text. Focus accents on inputs. Introduced Screen 01.
greenInk #1E7A52 Green as text/icon on white — the only green allowed on white (passes AA). Active nav label + icon.
darkPrimary #111111 Dark cards, screen headers, FAB background, primary button fill.
darkSecondary #1C1C1C Sub-cards on dark surfaces (e.g. stat tiles inside a dark summary card).

Surfaces

bgWarm #F2EDE4 Scaffold / page background. Warm cream. Never use pure white as scaffold.
bgCard #FFFFFF Card surface on bgWarm. White creates hierarchy without harshness.
activeBg #E8F5EE Active nav pill bg, success banner bg, selected state tint.

Text

textPrimary #111111 All body text, headings, labels on bgWarm and bgCard. 16.1:1 contrast on bgWarm.
textHint #8C8780 Placeholders, timestamps, captions, secondary labels. 3.6:1 on bgWarm — large text only.
textOnDark #FFFFFF All text placed on darkPrimary / darkSecondary surfaces.

Semantic

success #4DDE95 Confirmed actions, positive trends, MILKING / CALVED status.
warning #E8A020 Low yield badges, caution alerts, 1hr escalation, heat-alert initial state.
danger #D44C3A Critical alerts, 2hr+ escalation, notification badges, form errors.
border #D8D0C5 Card borders, input outlines, dividers. Warm-tinted to harmonise with bgWarm.

Alert escalation timeline

0 h

IoT detects temperature/behaviour anomaly. Farmer notified — push + in-app.

1 h

No farmer acknowledgement. Escalates to Farm Owner.

2 h

Farm Owner must act. Critical alert state — red border card, countdown timer visible.

6 h

Heat cycle confirmed in system. Status transitions to IN HEAT.

6 — Rules

Tap targets DO — every tappable element min 48×48dp. Farmers use phones outdoors, often with wet or gloved hands.
Font size floor DO — 14px absolute minimum. 16px for body. NEVER use 12px for anything the user must read to act on.
Line height DO — 1.7 for Devanagari body text. 1.5 minimum for Latin. The shirorekha (top bar) needs vertical clearance or letters collide.
brandGreen on white NEVER — #4DDE95 on white fails WCAG AA (2.5:1 ratio). Use it on #111 or on #E8F5EE pill backgrounds only.
String expansion DO — Hindi/Bhojpuri strings run 20–40% longer than English. NEVER fixed-width containers for translated text. Use flexible layouts everywhere.
RTL readiness DO — use EdgeInsetsDirectional and start/end everywhere. Urdu (Phase 4) will break all left/right hardcoded spacing.
Offline state DO — every screen must have an offline state. Show dark banner with wifi-off icon. A blank screen is never acceptable.
Scaffold background NEVER use #FFFFFF as scaffold background. Always bgWarm (#F2EDE4). Cards use white — the contrast is how visual hierarchy works.
Audit on edit DO — always show the amber audit banner when Farm Owner edits a Farmer's entry. Farm + Cow fields locked. Only volume/session editable.
Default locale Hindi (hi), not English. Fall back to hi for any unrecognised Indian locale. Bhojpuri code: bho → arb file: app_bho.arb.

main.dart — MaterialApp locale setup

MaterialApp(
  theme: vanixTheme(languageCode: 'hi'),
  localizationsDelegates: AppLocalizations.localizationsDelegates,
  supportedLocales: const [
    Locale('hi'),   // Hindi (default)
    Locale('bho'),  // Bhojpuri
    Locale('en'),   // English
  ],
  localeResolutionCallback: (locale, supported) {
    if (locale == null) return const Locale('hi');
    for (final s in supported) {
      if (s.languageCode == locale.languageCode) return s;
    }
    return const Locale('hi'); // fallback → Hindi
  },
)

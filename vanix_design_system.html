Brand — two-hue system only. No other hue.

Surfaces

Text

textPrimary #111111 All body text, headings, labels on bgWarm and bgCard. 16.1:1 contrast on bgWarm.

Semantic

border #D8D0C5 Card borders, input outlines, dividers. Warm-tinted to harmonise with bgWarm.

Card content — Title / Manager / View Details / CTA — every action card was trimmed to just four things: the title, the manager line, a small "View Details ›" text link (greenInk/greenDeep), and the card's own CTAs (Yes/No, single-acknowledge, etc.) rendered directly below — so the list stays scannable even with 14 card types in it. The full plain-language sensor description, the farm/belt/timing meta line, and (for the two evolving cards) the segmented heat-window bar and insemination form were all moved off the list card onto a dedicated Card Detail page, reached by tapping "View Details ›". The priority chip/corner badge stays on the list card — severity should be visible without a tap. One shared Card Detail template serves all 14 card types: header (back chevron + "Details"), title + priority badge, manager + meta, the full description, a temperature graph (last 10 days, today's reading highlighted as a red dot/bar against the baseline — mock data, no real 10-day sensor feed exists yet), a movement graph (same last-10-days treatment), then the card's CTAs repeated at the bottom so a farmer can act without leaving the detail view — resolving from either the list card or the detail page decrements the badge exactly once (HTML: the CTA's DOM node is physically moved into the detail page on open and moved back on close, so every existing click handler keeps working untouched; Flutter: the same CTA `Widget` is passed into both the list card and `CardDetailScreen`).
Priority system (P0–P3) — every card carries a chip signalling severity + channel, reusing only the locked palette (no new colours): P0 CRITICAL (Fever, Abortion/Pregnancy Loss, Fresh-Cow/Post-Calving Monitor) — no text chip (the full dangerBg/danger card tint + 4px left accent already says "critical"); instead a single small 20px dangerInk circle with a white "!" overlaps the card's top-right corner as a notification-style badge, replacing what used to be a stacked "P0 · CRITICAL" chip + separate "ESCALATED" chip (same red, said twice) — "Push + SMS · Immediate vet visit". P1 ACTIONABLE (Heat cycle → Insemination Window, Pregnancy Check Due) — solid dark-amber chip (warningInk fill, white text), full warnBg/warning tint, 4px left accent, "App notification · Schedule inseminator". P2 WARNING (Mastitis, Lameness, Ketosis) — outline chip (warnBg fill, warning border, warningInk text — same shape as history-timeline stage chips), neutral white/bgCard card with only a thin 2px warning left accent, "App inbox · Physical inspection". P3 WARNING (Rise in temperature, Herd Heat Stress, Calibration Complete — grouped under their own "Warnings" tab, separate from Needs Action) — outline chip (bgCard fill, border, textHint text), neutral white/bgCard card with no left accent, "App inbox · Info only". In dark mode all four tiers flatten to the #1C1C1C neutral surface with white titles, keeping their priority-coloured border/left-accent for severity — P2/P3 stay visually calm (thin/no accent) in both modes, and P0's corner badge stays legible on the dark surface without any changes needed. Chip text was trimmed from "P1 · ACTIONABLE" to a small bell/warning-triangle/info icon + "P1"/"P2"/"P3" — see "Icon + label" below.
Icon + label (farmer-friendly pass) — the app trends toward pictures over paragraphs, especially for the Farmer persona: an icon carries the meaning, a 1-2 word label backs it up (never a bare icon with no label, never a full sentence). Extends two patterns already in the app — the bottom nav (icon + 1-word label) and the icon-roundel + severity-colour convention used by Reminders/status dots. Applied so far to Events cards: the priority chip is now an icon (bell = P1, warning-triangle = P2, info = P3) + the bare tier code instead of "P1 · ACTIONABLE"; the manager line is a small person icon + name instead of "Manager: Name"; P2 inspection cards' Yes/No buttons are icon + one word ("✕ No" / "🚩 Flag it") instead of full sentences ("No, she's fine"); the single-acknowledge button (Proestrus/Herd Heat Stress/Calibration) gets a check icon. Still pending the same treatment: the P0 vet-flow cards' Yes/No sentences, Heat/Pregnancy/Gestation's remaining full-sentence buttons, and Milk Log (session → sun/moon icon instead of a text pill, litres → numeral + droplet icon) — tracked in CLAUDE.md's Rules section.
Card interaction patterns — five shapes reused across the 14 cards. (a) Vet-request flow (Fever, Abortion/Pregnancy Loss, Fresh-Cow Monitor — all P0): Yes/No confirm → No resolves to a false-alarm note; Yes opens a picker of the 3 onboarded vets (Dr. Sharma, Dr. Rao, Dr. Iyer — radio-style rows, first pre-selected, no email entry) + Send → "Vet appointment requested ✓ — Sent to [vet name]." (b) Inspection flow (Mastitis, Lameness, Ketosis — all P2): Yes/No confirm, no vet email — Yes resolves straight to "Flagged for inspection ✓ — check at next milking," no vet visit implied. (c) Single acknowledge (Rise in temperature, Herd Heat Stress, Calibration Complete — the Warnings tab): no diagnostic question, just one button since these are heads-up or zone-wide info rather than a diagnostic decision. Rise in temperature (formerly "Proestrus/early heat signs") is a temperature-only reading — its Card Detail graphs show a temperature spike but a flat/normal movement graph, since activity hasn't actually changed. (d) Single evolving card (Heat cycle, Gestation — both P1) — the SAME card automatically progresses through several phases as time/taps pass rather than firing a new alert at each stage. Heat: detection starts a real 24-hour clock immediately, visualised as a single SEGMENTED bar — three zones sized 25/50/25% (PRE 6h · OPTIMAL 12h · SUBOPT 6h) with zone labels above, the detection time at the left end and "+24h" at the right, and one phase-coloured fill travelling continuously across all three segments (never resetting at a boundary). After the farmer acknowledges "Yes, in heat," a "Call vet" button is available in EVERY phase: tapping it asks the farmer to pick one of the 3 onboarded vets, then opens the insemination log form (Method: Artificial / Conventional / IVF / Embryo Transfer, 2x2 selector + technician, prefilled with the chosen vet). In pre or suboptimal it first interposes a confirmation ("outside the optimal window — continue anyway?") before the vet picker. If the clock fully expires with nothing logged, the card auto-morphs to a direct choice: "Cow inseminated" opens the same form plus a time field, or "Insemination missed" logs it and resumes monitoring. The farmer's Yes/No tap on the initial confirm is just an acknowledgement — it does not pause or reset the clock; No stops it early as a false alarm. Sensor comparison basis: the cow's own 10-day rolling temperature/behaviour baseline (Cattle Health Logic v3.1, Block 4). In the HTML mockup 1 simulated hour = 4 real seconds (24h plays out in 96s; the walkthrough copy runs at 2s/h) purely so the phase transitions are demoable live — production wires this to the real backend peak_timestamp instead. Gestation: 3-month vet check (single ack) → 6-month vet check (single ack) → 9-month "call your vet for delivery" (same 3-vet picker as the P0 cards) → "was the delivery successful?" (Yes → notes-log form that resolves and triggers the Milking notification below; No → records the loss, re-alerts the vet, and the cow does not enter milking) — only the final Yes/No tap calls resolveEvent(). (e) Notification-with-defer (Milking notification, End-of-lactation check-in — both P1): two buttons where only one resolves the card. Milking notification — "[Cow] is now in her lactation period (250 days) — add to milking list?" → "Remind me later" shows a 24h-reminder note and loops back to the same question without touching the badge; "Yes, add" resolves to "Added to Milk Log ✓." End-of-lactation check-in — "Is [Cow] still milking, or has she entered her resting period?" → "Still milking" shows a 10-day recheck note and loops back without touching the badge; "Entered resting period" resolves and starts the 60-day dry/cool-down. Pregnancy Check Due is a plain Yes/No card, simplified this session to a farmer self-confirm ("Confirm — pregnant") rather than requiring vet confirmation.
Full-screen heat alert carousel — tapping "View full cycle ›" first opens a full-bleed, MyGate-style takeover screen (follows light/dark mode — bgWarm surface in light, darkPrimary/darkSecond gradient in dark) that is now a SWIPEABLE CAROUSEL of every cow currently in heat (demo: Gauri, Mohini, Dhauli). Chrome: ✕ top-left, "N of 3" counter top-right, ‹/› arrow buttons at the card's vertical midline, swipe left/right also navigates, shared "View in app instead ›" link at the bottom. Each card: farm/belt + detection time, tinted circular avatar, title, a one-line REASONING sentence explaining why the system flagged her, a last-10-readings temperature sparkline (warning-coloured line on a translucent panel with min/max °C labels), a 10-bar movement/activity mini-graph (greenDeep bars), then the actions — "Yes, in heat" as the PRIMARY button on top (greenDeep fill with dark #111 text, per the locked CTA token rule — not warningInk) and outline "No" below. Actioning a card morphs it to "Acknowledged ✓" and auto-advances to the next unresolved card; when all cards are actioned it falls through to the walkthrough bottom sheet, seeded with the FIRST card's (Gauri's) decision since she is the cow the walkthrough narrates. Dismissing via ✕ or the link at any point drops into the bottom sheet with a "restricted" version of the heat step (title + one-line question only, no detail rows or bar until Yes/No is tapped there).
Full-cycle walkthrough — the bottom sheet that follows steps through the entire breeding/lactation year for one cow in 8 sequential steps: Heat detected (the real evolving Heat card, own 24h ticker) → 21-day pregnancy watch (narrated interstitial, Continue) → Pregnancy check due (real card, farmer self-confirm) → 9-month gestation (narrated interstitial mentioning the 3/6/9-month checks, Continue) → 9-month vet check / delivery confirmed (real card) → Milking notification (new — "lactation period (250 days) begins," Remind-me-later/Yes-add) → End-of-lactation check-in (new — Still-milking loops/Entered-resting-period) → Dry/resting period (info-only, final step — "60-day dry period," Finish → "Cycle complete — ready for her next heat cycle"). Chrome: "Step N of 8" label + "Skip walkthrough" link on every sheet. Tapping a negative outcome (No at the heat step, Not pregnant at the pregnancy-check step) ends the walkthrough early with an interrupted-cycle message rather than forcing the happy path. Critically, the whole sequence runs on its own local state — it never touches the live `evOpen`/badge counter or resolves the real inline cards, so running it doesn't consume any of the farmer's actual pending alerts.

6 — Rules

Tap targets DO — every tappable element min 48×48dp. Farmers use phones outdoors, often with wet or gloved hands.
Font size floor DO — 14px absolute minimum. 16px for body. NEVER use 12px for anything the user must read to act on.
Line height DO — 1.7 for Devanagari body text. 1.5 minimum for Latin. The shirorekha (top bar) needs vertical clearance or letters collide.
brandGreen on white NEVER — #4DDE95 on white fails WCAG AA (2.5:1 ratio). Use it on #111 or on #E8F5EE pill backgrounds only.
String expansion DO — Hindi/Bhojpuri strings run 20–40% longer than English. NEVER fixed-width containers for translated text. Use flexible layouts everywhere.
RTL readiness DO — use EdgeInsetsDirectional and start/end everywhere. Urdu (Phase 4) will break all left/right hardcoded spacing.
Offline state DO — every screen must have an offline state. Show dark banner with wifi-off icon. A blank screen is never acceptable.
Scaffold background NEVER use #FFFFFF as scaffold background. Always bgWarm (#F2EDE4). Cards use white — the contrast is how visual hierarchy works.
Audit on edit DO — always show the amber audit banner when Farm Owner edits a Farmer's entry. Farm + Cow fields locked. Only volume/session editable.
Default locale Hindi (hi), not English. Fall back to hi for any unrecognised Indian locale. Bhojpuri code: bho → arb file: app_bho.arb.

    }
    return const Locale('hi'); // fallback → Hindi
  },
